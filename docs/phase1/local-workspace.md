# F1.1 本地工作空间

## 现在能做什么

本地核心已经有独立的工作空间骨架，可以读取指定范围内的原件，把文件复制为只读证据快照，并备份、校验和恢复状态目录。它不会启动服务器，也不会改写鸿喜达原件。

## 配置顺序

工作空间路径按以下顺序取值，前面的覆盖后面的：

1. 调用时显式传入的路径；
2. 环境变量；
3. JSON 配置文件；
4. 当前用户目录下的 `FoxWork`。

支持三个环境变量：

| 变量 | 用途 |
|:---|:---|
| `BRAND_OS_CONFIG` | 指定 JSON 配置文件 |
| `BRAND_OS_WORKSPACE` | 指定工作空间根目录 |
| `BRAND_OS_SOURCE_ROOTS` | 指定允许读取的原件根目录，多个路径用系统路径分隔符连接 |

配置文件只接受 `workspace_root` 和 `source_roots`。密钥、密码和模型 Token 不写进该文件；出现额外字段时程序直接拒绝加载。可从 `config/brand-os.example.json` 开始配置。

## 目录分区

初始化后，程序只在工作空间下创建 `.fox`：

```text
.fox/
├── workspace.json     # 不含密钥的目录说明
├── state/             # 正式状态与后续 SQLite 权威库
├── evidence/sha256/   # 按内容哈希保存的只读证据快照
├── backups/           # 经清单和哈希校验的状态备份
├── derived/           # 可重建索引与摘要
└── runtime/           # 可删除的会话、缓存与临时产物
```

`.fox` 下的目录权限固定为当前用户可读写。证据快照提交后设为当前用户只读。原件路径只用于读取，不会被改权限、重命名或覆盖。

## 路径与完整性规则

- 原件必须位于 `source_roots` 内，越界路径直接拒绝。
- 原件符号链接直接拒绝，避免借链接跳出允许目录。
- `.fox` 内的状态、备份和派生文件不能重新冒充原件导入。
- 快照使用完整 SHA-256 作为文件名；已有同哈希快照不会重复写入。
- 打开快照前重新计算哈希，发现篡改就停止。
- 状态备份不包含 `runtime` 和 `derived`；恢复前逐文件校验清单。

F1.1 的目录备份只用于非数据库状态。F1.2 已增加 SQLite 在线备份与恢复；普通目录备份遇到数据库、WAL 或 SHM 文件会停止，不能复制正在写入的数据库文件。

## 当前限制

- SQLite Schema、事件、投影和在线备份已完成；鸿喜达资料幂等导入仍属于 F1.3。
- 还没有 CLI/MCP 或界面，这些属于 F1.8 和 F1.9。
- 当前没有把远程 14GB 资料复制到本地，也没有处理无权限目录。
- OpenWork 尚未接入；删除任何 OpenWork/OpenCode 运行态都不会影响这里的状态和证据。
