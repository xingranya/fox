# F1.6 当前状态查询与证据回源

## 结果

系统现在能直接回答三类问题：当前有哪些决定、还有哪些开放问题、某条结论依据什么。

默认查询只返回当前生效的内容。已被替代、已驳回、已经过期或尚未生效的内容仍保留在历史里，但不会混进当前任务。需要复盘时，可以显式打开历史查询。

本次新增 `evidence-query.v1`，由独立的 `EvidenceQueryPort` 对外提供读取能力。查询不会写事件、改投影或批准 Proposal。

## 有效性怎么判断

决定和开放问题沿用 Proposal 生命周期，再叠加人工批准的有效期：

| 状态 | 含义 | 默认进入当前查询 |
|:---|:---|:---:|
| `current` | Fox 已批准，当前投影中存在，且没有过期 | 是 |
| `superseded` | 已由新的批准内容替代 | 否 |
| `archived` | Proposal 已被驳回，只保留审计记录 | 否 |
| `expired` | 已超过 `valid_until` | 否 |
| `scheduled` | 还没到 `valid_from` | 否 |
| `unknown` | 生命周期、时间或引用不足，不能确认 | 否 |

`valid_from` 和 `valid_until` 必须带时区，结束时间必须晚于开始时间。有效期只随 Fox 批准的 Proposal 进入正式投影，AI 不能单独改有效期。

来源版本另有一层状态：`current`、`superseded`、`archived` 或 `unknown`。文件有了新版本，不会让旧版本的历史证据失效；旧证据仍按原 SHA-256 回源，只是明确提示它已经不是来源的当前版本。

## 回到原件或会议原话

当前支持两种稳定引用：

```text
meeting:<meeting_id>#<segment_id>
source-version:<source_version_id>#<locator>
```

解析结果会返回来源版本 ID、SHA-256、相对路径、保密级别和 `evidence://sha256/<sha256>` 打开引用。会议引用还会返回原话、说话人、发言时间或时间段、会议时间和上下文。

一条重要结论只有同时满足以下条件，证据状态才是 `confirmed`：

1. 有 Fox 的批准事件；
2. 至少有一条可识别的证据引用；
3. 所有引用都能解析到固定来源版本；
4. 会议证据有原话、说话人、时间位置和已核验来源。

决定已经批准但证据引用不完整时，结果会分开写成“决定已确认、证据未确认”。旧的自由格式引用不会被猜成某个文件或某段会议。

## 关系查询

查询支持以下 11 种关系：

- 来源于、提出于；
- 支持、反对、冲突；
- 适用、批准；
- 替代、依赖；
- 回答、待确认。

显式登记的工作层关系仍是工作层，不会因为可查询就升级为决定。系统还会从已经存在的权威记录中派生会议提出、人工批准、Proposal 替代、来源版本替代和会议冲突关系，并注明关系来源。

关系端点如果已被替代、归档或过期，默认不返回。端点不存在或证据无法解析时，关系仍可见，但状态是 `unknown` 或 `unconfirmed`，方便补证，不制造确定性。
