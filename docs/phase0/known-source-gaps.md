## 已声明但未核验的数据

- 需求说明中关于 V5 原件数量、内容分段数量或历史处理规模的描述，只作为规划输入保存。
- 在获得对应原件、Manifest 和人工确认前，其证据状态统一为 `declared_unverified`。
- `declared_unverified` 不得进入正式事实投影，也不得用于对领导宣称系统已经完成真实全量接入。

## 解除缺口所需最小输入

当前已获得远程资料入口。后续还需要以下最小输入：

1. 处理当前无权限目录，或明确将其排除在首轮全量对账之外。
2. 明确远程资料中允许进入外部模型的级别；当前默认只做本地读取。
3. F1.5 完成 Proposal 生命周期后，选择一组 `confirmed_by_fox` 记录做首批人工复核。
4. BrandBench 第二轮人工评分已完成；Phase 2 继续做真实任务评审，不再阻断 Phase 1。

远程全量对账、外发授权和会议人工校对仍按缺口管理，但不阻断已经通过的 Phase 0 技术实施就绪门。Phase 1 只使用当前已授权、已登记的来源。
