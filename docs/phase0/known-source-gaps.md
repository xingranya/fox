# Phase 0 已知资料缺口

## 当前结论

当前授权材料只能证明已完成 `example/` 的只读盘点与本地解析，不能证明完整鸿日/V5 基线已获得或完成对账。需求说明中出现的资料数量与分段数量均按 `declared_unverified` 处理，不转写为已验证事实。

## 缺口清单

| 缺口 ID | 状态 | 缺失内容 | 当前影响 | 允许动作 | 禁止表述 |
|:---|:---|:---|:---|:---|:---|
| `GAP-001` | `KNOWN_SOURCE_GAP` | 完整鸿日/V5 原件集合及稳定目录 | 无法验证完整数量、版本、重复、失效与替代关系 | 保留缺口；等待 Fox 提供只读位置或受控副本 | “已完成 V5 全量对账” |
| `GAP-002` | `KNOWN_SOURCE_GAP` | 各原件的来源角色、创建者、适用阶段与当前有效性确认 | 无法建立可信的 current/superseded/archived 投影 | 先生成待确认登记，不自动推断 | “这些资料都是当前有效版本” |
| `GAP-003` | `KNOWN_SOURCE_GAP` | 鸿日已批准事实、决定、约束和外部承诺的人工确认事件 | 无法把规划材料内容写入正式状态 | 只形成逐项 Proposal，等待 Fox 确认 | “PPT 中的内容就是已批准决定” |
| `GAP-004` | `KNOWN_SOURCE_GAP` | 会议音频与人工校对转写、说话人、时间位置 | 无法把模型转写作为正式原话证据 | 保留模型候选转写及不确定性 | “模型转写等于会议原话” |
| `GAP-005` | `KNOWN_SOURCE_GAP` | 普通聊天 BrandBench 基线与 Fox 匿名评分 | F0.6 和 F0.7 阶段门不能通过 | 先准备评分表与匿名化流程，等待 Fox 评分 | “品牌质量已达到基线” |

## 已声明但未核验的数据

- 需求说明中关于 V5 原件数量、内容分段数量或历史处理规模的描述，只作为规划输入保存。
- 在获得对应原件、Manifest 和人工确认前，其证据状态统一为 `declared_unverified`。
- `declared_unverified` 不得进入正式事实投影，也不得用于对领导宣称系统已经完成真实全量接入。

## 解除缺口所需最小输入

Fox 后续只需提供以下信息，不需要先整理全部内容：

1. 完整鸿日资料的只读目录位置，或一个受控副本目录。
2. 哪些文件允许本地模型读取，哪些文件禁止进入任何 AI 上下文。
3. 至少一项已批准决定或约束及其可回源证据，用于建立第一条人工确认事件。
4. 对 BrandBench 匿名候选进行一次人工评分。

在以上输入到位前，系统继续使用脱敏样本完成 Schema、Fixture、测试和 OpenWork 接入准备，但不越过 Phase 0 实施就绪门。
