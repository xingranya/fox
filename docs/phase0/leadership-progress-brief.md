# Brand Project OS 阶段成果简报

> 日期：2026-07-22；范围：使用当前已登记的本地样本和远程鸿喜达 9 份当前来源，不宣称 14GB 远程资料已经逐文件完成全量对账。本文已同步同日后续的 OpenWork 单一客户端决定。

## 已完成

- 建立 5 类本地只读样本处理链，覆盖 PDF、DOC、DOCX、PPTX 和 MP3。
- 完成来源哈希、稳定本地 ID、来源角色、P2/P3 分级、Office/PDF 正文提取和本地音频候选转写。
- 建立会议语义契约，区分观点、偏好、假设、选项、倾向、目标日期、决定候选、约束候选、行动候选和开放问题。
- 冻结品牌 Agent 宪法与探索、评估、决策、执行四种工作模式；只有 Fox 可以切换模式或批准业务状态。
- 建立 16 个黄金用例，覆盖冷启动、会议解释、证据回查、增量更新、策略探索、执行、模型交接和品牌匿名评审。
- 七项一票否决已进入机器校验：不得虚构事实、升级讨论、升级暂定日期、使用废案、失去证据、越权改状态或在探索中强行唯一答案。
- 完成本地核心端口、Task Packet、Proposal 和 OpenWork 客户端适配契约。
- 完成两轮 BrandBench 人工评审。第二轮采用稿 23/30，自然中文 4/5，没有一票否决，Phase 0 已通过。

## OpenWork 路线

OpenWork 已确定为公司唯一员工客户端基础，但不是业务真相数据库。它负责展示当前状态、证据、Agent 运行和 Proposal；正式状态、批准与模式切换仍由自有领域核心处理。客户端运行数据被清理后，项目状态和证据不能丢失。

当前顺序是先完成 F1.9 的离线、安全、品牌和单安装包收口，再用 F1.10 接入当前状态、证据、Task Packet、会议 Proposal 和 Fox 确认，完成桌面旅程与黄金回归。
