# BrandBench Phase 0 匿名评审指引

## 当前状态

评分框架、六个质量维度、匿名规则和机器校验已经建立。首轮失败基线和第二轮人性化结果均已由 Fox 完成人工评分；第二轮采用候选通过 Phase 0 质量门。

## 评分维度

每个匿名候选按 1-5 分分别评审：

1. 战略锋利度：是否形成清晰、有取舍、非模板化的战略判断。
2. 自然中文：是否自然、具体、像真实工作语言，而不是模型腔。
3. 消费者真实：是否建立在真实生活感受与行为上。
4. 记忆性：是否存在可被记住和复述的核心表达。
5. 产品咬合度：是否与已知产品证据紧密连接，避免空泛品牌话术。
6. 证据纪律：是否区分事实、假设和未知，重要结论是否能回源。

## 正式评分时的操作

1. 使用同一任务分别生成“无项目状态层普通聊天”候选和“Brand Project OS Task Packet”候选。
2. 删除模型、系统和生成顺序标识，将候选随机映射为 A、B。
3. 先运行七项一票否决硬门；硬门失败的候选不进入品牌质量排名。
4. Fox 独立完成全部 12 个评分单元并写一句必要理由。
5. 评分后才填写 `reviewed_by`、`confirmed_at`，并把 `review_status` 改为 `completed`。
6. 保存真实候选、匿名映射和评分到本地受控目录，不提交可能含业务原文的内容。

## Phase 0 准入规则

- 七项一票否决发生数必须为 0。
- 六维中位数不得低于普通聊天基线。
- 至少三个维度需要出现可见改善。
- 当前样本不足以证明统计显著，只能报告本次评审结果。

采用候选还必须由 Fox 明确选择，并明确回答一票否决问题。Task Packet 是内部上下文与证据结构，不要求对外文字复述它的目录。

空白评分模板位于 `fixtures/phase0/brandbench-review-template.json`。正式候选和匿名映射只放在 `.work/phase0/`；仓库仅保存去除原文与匿名映射后的评分结果。
