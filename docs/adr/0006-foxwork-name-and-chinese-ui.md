# ADR-0006：FoxWork 发行名与简体中文界面

- 状态：已接受
- 日期：2026-07-23
- 决策人：Fox
- 影响范围：客户端名称、图标、AppID、协议、更新源、安装包、界面文案和验收
- 补充：[ADR-0004](0004-openwork-single-client.md) 与 [ADR-0005](0005-single-client-server-authority.md)

## 决策

1. 公司定制版 OpenWork 的发行名固定为 FoxWork。员工安装、打开和更新的始终是 FoxWork，不再使用“最终发行名待定”的表述。
2. Brand Project OS Service 保留为服务器业务能力名称。它不形成第二个客户端，也不要求员工单独安装或管理。
3. FoxWork 面向员工的界面、提示、错误信息、安装说明和更新文案只使用简体中文，不提供语言切换。
4. 运行时可以保留上游源码中的其他语言资源用于维护和追溯，但正式构建不得加载、展示或回退到英文及其他语言。缺失中文文案时应在测试或构建阶段失败，不能把英文或 i18n 键名交给员工。
5. FoxWork 使用公司控制的名称、图标、Bundle ID/AppID、深链协议和更新源。迁移兼容可以识别旧标识，但新建数据、协议注册和发行产物不得继续使用上游或临时品牌。

## 实施位置

- F3.2：新增的登录、服务器连接、项目选择、离线、证书和撤权界面只显示中文。
- F3.3-F3.5：新增的业务状态、人工确认和本机权限界面继续执行同一约束。
- F3.13：检查完整客户端、安装包和运行产物，确认发行名为 FoxWork、语言固定为简体中文、没有可见语言切换或英文回退。
- F4.8：用 FoxWork 的公司标识完成签名、公证、内部更新、灰度和回滚验收。

## 验收

- 应用名、窗口、菜单、安装包、协议注册和更新界面统一显示 FoxWork。
- 首次启动、已有用户升级、离线、登录失败和服务异常时仍显示简体中文。
- 设置中没有语言选择入口；修改旧的本机语言偏好不能让正式构建切换到其他语言。
- 自动化测试枚举新增员工可见文案，产物扫描阻断旧品牌、英文回退和未解析的 i18n 键。
- 这项决定不改变单一客户端、服务器权威、人工确认和本机权限边界。
