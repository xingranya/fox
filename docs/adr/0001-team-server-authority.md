# ADR-0001：团队服务器权威架构作为远期候选

- 日期：2026-07-13
- 取代日期：2026-07-22
- 影响范围：未来团队部署、多人身份、权限、并发、远程访问和灾备
- 当前生效决策：[ADR-0005：单一客户端与服务器权威服务](0005-single-client-server-authority.md)

> 本文记录第一次 rescope 时为何暂缓服务器。Fox 已在 2026-07-22 明确团队部署和单一客户端目标，正文不再代表当前实施顺序。

## 背景

团队服务器、PostgreSQL、S3、OIDC、RLS、Outbox、远程 MCP 和灾备方案已经完成较深入研究。这些能力能够解决多人并发、统一权限、远程访问和服务恢复，但它们服务的是未来团队化问题，不是当前第一验证目标。

当前第一用户是 Fox，第一验证项目是鸿日，工作空间位于 `/Users/fox/work`。现阶段首先要证明系统能否正确理解长期品牌项目、增量解释会议、保持证据回源、支持探索与执行两种协议，并在模型切换后维持一致项目认知。提前建设完整服务器会显著推迟这一验证，也可能把尚未确认的团队需求误写成产品前提。

## 决策

1. 团队服务器方案不进入当前鸿日本地 MVP 的实现前置条件，不得据此要求先建设 PostgreSQL、S3、OIDC、RLS、Outbox、完整 Web/PWA、高可用或生产灾备。
2. 既有服务器研究完整保留在[团队服务器架构](../plan/team-server-architecture.md)和[数据一致性与可靠性计划](../plan/data-consistency-and-reliability.md)中，但必须明确标记为 `future-candidate`，不得使用“当前已批准”“首发生产形态”或同义表述。
3. 当前 MVP 的本地权威边界、单写入模型和未来迁移约束由 ADR-0003 定义。SQLite 和本地内容寻址证据库可以承担鸿日单用户验证，不因此形成与服务器的双主。
4. 只有鸿日试点证明产品价值后，才评估是否需要团队账户、远程共享、并发审批、中心化对象存储和服务级恢复。评估必须基于真实使用数据，不以既有架构完整度替代需求确认。
5. 若后续决定服务器化，应新建一份“团队服务器正式采用”ADR。届时本 ADR 中已研究的单一权威、客户端统一 API、外部组件可替换和可恢复迁移原则作为候选输入，而不是自动生效的决定。

## 未来候选约束

若服务器路线通过复审，候选架构仍应满足：

- 标准 PostgreSQL 是服务器正式状态、领域事件、人工审批和审计的唯一可写数据库。
- 原始文件进入团队控制的 S3 兼容对象存储，按 SHA-256 寻址并启用版本保护。
- 桌面、Web、CLI、MCP、Skills 和自动化调用同一版本化应用 API，不直接写正式数据库或对象存储。
- Zvec、Open Notebook、Nubase Memory、FlowLong、Dify、Agent 会话、缓存和模型摘要均为运行态、派生态或协调态，不覆盖正式状态。
- 从本地 MVP 迁移时采用一次性、可校验的单向切换；稳定 ID、事件、审批和证据哈希必须保留，不允许长期本地/服务器双写。

## 后果

### 正面

- 把验证顺序拉回“先证明 Fox 提效，再决定团队化”，减少基础设施先行造成的范围漂移。
- 保留已有服务器研究，避免未来确认团队需求时重新设计。
- 当前实现可围绕真实鸿日资料、会议解释、Task Packet 和人工确认形成更短的纵向切片。

### 代价

- 当前 MVP 不提供多人账户、远程协作、服务端高可用、中心化权限或生产级灾备承诺。
- 未来服务器化需要迁移工具、数据对账和新的正式采用 ADR，不能把本地 SQLite 直接暴露为团队数据库。

## 当前被拒绝的做法

- 为了“以后团队会用”而先完成全部服务器基础设施。
- 把历史上的 FoxWork 团队文件协作需求直接合并为鸿日 MVP 的已确认范围。
- 在本地验证尚未通过前，以在线率、并发量或 RPO/RTO 作为第一阶段产品价值指标。
- 同时维护本地正式库和服务器正式库，并依赖双向同步解决冲突。

## 复审门

鸿日试点至少提供以下证据后，才允许复审服务器化：

1. Fox 连续使用完成一轮真实会议、状态更新、策略探索和执行交付。
2. 冷启动、会议分类、证据回源、增量 Proposal、模型切换和品牌质量金标达到基线。
3. 明确出现多人同时使用、远程共享、中心权限、自动化常驻或本机可靠性无法满足的真实需求。
4. 能回答哪些数据必须中心化、哪些能力应继续本地优先，以及是否与 FoxWork 团队协作需求合并。
5. 已制定从本地权威到服务器权威的单向迁移、校验、回滚和停止双写方案。
