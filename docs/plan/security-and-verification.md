# 安全与验收计划

## 安全目标

1. 团队成员、AI 和自动化看到同一权威状态，但只获得任务所需的最小数据与动作 Scope。
2. 任何入口都不能绕过人工批准、租户隔离、并发校验、幂等和审计。
3. PostgreSQL、S3 原件、备份、模型/工作流和客户端缓存按不同信任边界治理。
4. 派生组件失效或被入侵时，正式状态和原件仍完整，核心服务可降级运行。
5. 稳定性由监控、故障注入和独立恢复演练证明，不由单机 Docker Compose 或配置声明代替。

本计划与[团队服务器架构](team-server-architecture.md)、[数据一致性与可靠性计划](data-consistency-and-reliability.md)和[前端与 AI 访问规划](frontend-and-ai-access.md)共同构成 Phase 1-6 的发布门。

## 信任区与网络边界

| 信任区 | 允许入口 | 禁止事项 | 主要控制 |
|:---|:---|:---|:---|
| 公网边缘 | 仅 HTTPS 443 | 暴露 PostgreSQL、对象存储管理端、Zvec、Dify/Notebook/FlowLong 管理端和监控控制面 | TLS、WAF/限流、请求大小、`request_id`、安全响应头 |
| Web/PWA | OIDC 登录后的应用 API | 数据库凭据、Dify Key、长期服务 Token、离线批准 | 安全 Cookie、CSRF、CSP、MFA、缓存分级 |
| CLI/MCP/Skills | OAuth/设备授权/PKCE 或最小权限服务凭据 | 明文保存刷新令牌、直连数据库、任意工具调用 | 系统钥匙串、Scope、工具允许列表、撤权和审计 |
| 应用/Worker 私网 | PostgreSQL、S3、受控外部适配器 | 共享超级用户、跨组件共用管理员密钥 | 独立角色、短期凭据、出口白名单、mTLS/私网 |
| 数据平面 | PostgreSQL、正式对象区、备份域 | 公网直达、同机备份充当灾备、应用自动迁移生产 Schema | RLS、加密、版本控制、PITR、对象锁、显式迁移 |
| 外部/派生组件 | 经版本化端口和最小 Task Packet | 正式状态写入、批准、完整数据镜像、无审计外发 | 隔离账号/数据库、Schema 校验、功能开关、NoOp 和对账 |

运维入口使用 VPN、Tailscale、堡垒机或云控制面并启用 MFA。日志不得记录 Token、密码、完整 P2/P3 原文、未脱敏 Prompt、Cookie、对象签名 URL 或模型密钥。

## 身份、角色与权限

### 身份规则

- Web 使用 OIDC 授权码+PKCE、服务端安全 Cookie、`Secure`/`HttpOnly`/合适 `SameSite` 和 CSRF 防护。
- 高风险批准、成员管理、密钥轮换和生产恢复支持 MFA 或短时重新认证。
- CLI 使用设备授权或 PKCE 本机回调，刷新凭据进入系统钥匙串；禁止把 Token 写入项目文件或 Skill。
- 远程 MCP 优先 OAuth。受控试点使用 Bearer 时，必须来自环境变量、绑定服务账号、项目、工具 Scope 和过期时间。
- 服务账号不共享，不使用永久全局 Token；创建、轮换、撤销和使用均进入审计。
- 撤销成员或服务账号后，Web 会话、CLI Token、MCP OAuth 和自动化凭据必须在定义的撤权目标内失效。

|:---|:---:|:---:|:---:|:---:|:---:|:---:|
| 组织所有者 | 是 | 按项目角色 | 是 | 仅同时具备批准 Scope 时 | 是 | 受控 |
| 项目负责人 | 是 | 是 | 是 | 是 | 项目范围 | 否 |
| 项目管理员 | 是 | 是 | 是 | 默认否 | 项目范围 | 否 |
| 审批人 | 是 | 按授权 | 是 | 仅授权对象/动作 | 否 | 否 |
| 编辑者 | 是 | 是 | 是 | 否 | 否 | 否 |
| 查看者 | 是 | 否 | 否 | 否 | 否 | 否 |
| 外部访客 | 限资源/限时间 | 默认否 | 仅评论型候选 | 否 | 否 | 否 |

## RLS 与租户隔离

- 所有业务表包含 `workspace_id`；项目资源同时包含 `project_id`，使用复合外键阻止跨工作空间引用。
- 生产业务表启用并强制 RLS；运行时角色不是表所有者，也没有 `BYPASSRLS`。
- 每个事务使用 `SET LOCAL` 注入主体、工作空间、项目和 Scope，事务结束自动清除，避免连接池复用串租户。
- 迁移、API、Outbox Worker、索引 Worker、备份和只读审计使用不同数据库角色。
- 应用层先做角色、Scope、对象、状态机和保密策略校验，RLS 只作纵深防御。
- 跨租户测试必须同时覆盖 API、直接数据库访问、ID 枚举、搜索索引、对象键、缓存、Task Packet、导出和外部工作流。

## 数据分级与外发

| 等级 | 示例 | 默认存储/处理 | 外发规则 |
|:---|:---|:---|:---|
| P0 公开 | 已公开网页和品牌资料 | 团队服务，可进入登记索引 | 经任务授权可调用登记过的服务 |
| P1 内部 | 普通会议纪要和内部策略 | 团队控制的 PostgreSQL/S3 | 外部模型/插件需任务级授权和留痕 |
| P2 客户机密 | 未发布产品、报价、渠道反馈 | 加密存储，最小项目权限 | 默认禁止；仅批准的自托管处理或脱敏副本 |
| P3 高敏 | 合同、个人信息、密钥、资质原件 | 专项加密、最小授权，不进通用 AI 检索 | 禁止进入通用 AI、Dify 插件或外部抓取链路 |

自托管 Dify、Open Notebook 或模型网关不等于“没有外发”。模型提供商、Embedding、STT/TTS、插件、HTTP 节点、网页抓取、遥测和对象存储地域都必须登记。每次 AI 运行保存数据级别、提供商、任务范围、人工授权、工作流/模型版本和最终 Proposal，不保存不必要的完整原文副本。

## 原始文件与不可信输入

- 上传先进入隔离区，校验大小、SHA-256、MIME/魔数、文件类型、恶意内容和解压限制，再转为内容寻址不可变对象。
- 临时对象、未校验文件和外部 URL 不能成为正式证据。
- 文档、会议、网页和 Notebook 内容全部视为不可信数据；其中的“系统指令”“FINAL”“批准”或工具请求没有权限含义。
- 解析器在资源受限沙箱运行，限制 CPU、内存、文件数量、递归深度、网络和执行权限。
- 对象访问使用短期签名 URL、租户/项目绑定和最小方法；不得把永久公开 URL 写入索引或 Prompt。
- 正式版本用墓碑撤销，不原地覆盖；对象存储开启版本控制、服务端加密和适当的删除保护/对象锁。

## 远程 MCP、CLI、Skills 与 Dify

### 远程 MCP

- 只允许当前状态、Task Packet、会议、证据搜索/回源、决定、开放问题、行动和 `proposal_create`。
- 禁止 `approve/reject`、任意 SQL、成员管理、密钥读取、原件硬删除、Dify 任意编辑和第三方管理工具。
- Streamable HTTP 和 stdio 代理共享 OAuth/Scope、速率限制、超时、请求标识和审计；代理不缓存业务事实。
- 工具参数必须经过 JSON Schema、租户上下文、对象 Scope 和保密级别校验，工具结果按最小披露返回。

### CLI 与 Skills

- CLI 交互式管理动作显示目标、旧新差异、影响范围和版本，要求重新认证和明确确认；仍不能让 AI 服务账号批准。
- Skills 只保存可复用流程、工具顺序、Schema 和失败处理，不保存业务事实、聊天摘要、密钥或数据库连接串。
- AGENTS 保存长期团队规则，Skills 保存可重复工作法，MCP 提供实时状态，CLI 服务具名人员和脚本，四者不能互相替代。

### Dify

- 后端是 Dify 唯一调用方；浏览器、MCP、CLI 和 Skill 不持有 Dify API Key。
- 输入是最小 Task Packet，输出必须通过固定 JSON Schema，只能生成 Proposal。
- 工作流键、DSL 导出、Prompt、模型、插件、超时、重试、运行 ID、数据级别、成本和退出方案均留痕。
- Dify 停止、超时、取消或返回错误 Schema 时，权威状态不变，核心查询与人工审批保持可用。
- 首个 POC 仅单内部 Workspace、后端 API；未经 Dify 书面授权，不运行源码多租户、移除/修改其前端品牌或对外分发。

### FlowLong

- 只协调复杂人工任务，不负责 AI 推理和正式状态写入。
- 未获得部署/嵌入/分发形态的书面许可结论前，不用真实数据运行 POC。
- 强制关闭 AI 审批和超时自动通过；回调重新验证签名、操作者映射、工作空间、项目、流程版本、Proposal 版本和幂等键。

## 威胁与必测防护

| 威胁 | 防护 | 必测结果 |
|:---|:---|:---|
| 跨租户 ID 枚举或 RLS 旁路 | 应用 Scope+强制 RLS+复合外键 | API、SQL、搜索、对象、导出均读不到其他项目 |
| 重放、双击和并发覆盖 | 幂等键、请求摘要、聚合版本、行锁 | 100 次重试只产生一个事件；冲突为可处理 409 |
| 提示注入和伪批准 | 不可信输入、工具允许列表、AI 无批准能力 | 文档中的指令无法调用禁用工具或改变正式状态 |
| 外部回调伪造/重放 | 签名、来源白名单、Inbox、目标版本 | 重复/乱序/旧版本回调不产生重复或覆盖 |
| 对象替换或回源断裂 | SHA-256、版本、对象锁、定期对账 | 改名不改变身份；内容变化形成新版本；缺失立即告警 |
| Token/密钥泄漏 | 短期凭据、钥匙串/密钥管理、日志脱敏、轮换 | 旧凭据撤销后不可用，仓库/日志/前端无密钥 |
| 派生层污染权威状态 | 单向 Outbox、Schema 校验、Proposal 门 | 删除 Dify/Zvec/Notebook/FlowLong 数据后权威状态完整 |
| Worker 崩溃与重复投递 | Outbox/Inbox、有限重试、死信 | 外部成功确认前崩溃后重放不产生重复结果 |
| 备份不可恢复 | PITR、对象版本、独立故障域、定期演练 | 隔离环境在 RPO/RTO 内恢复并通过金标与权限校验 |
| 离线客户端形成分叉 | 只读快照、草稿、基础版本和冲突处理 | 断网不能批准；联网后服务器版本变化则进入人工冲突 |
| 依赖/镜像供应链风险 | 固定版本/摘要、SBOM、签名、许可证和扫描 | 未登记或高危依赖阻断发布；升级先过全套回归 |

## 测试分层

| 层级 | 核心用例 | 通过标准 |
|:---|:---|:---|
| 领域单元 | 状态升级、授权、替代、`not_implies`、日期性质 | 非法迁移全部拒绝，AI 无批准路径 |
| Schema/契约 | JSON Schema、OpenAPI、MCP、端口、错误码、幂等、版本 | 所有适配器通过共享 contract test |
| PostgreSQL | 迁移、事务回滚、唯一约束、行锁、RLS、连接池、序列化重试 | 无丢失更新、串租户、部分提交或自动生产迁移 |
| Outbox/Worker | 原子提交、并发领取、重复投递、乱序、死信、世代切换 | 至少一次投递但业务结果幂等，故障后可追平 |
| S3/对象 | 隔离、哈希、MIME、版本、孤立对象、对象锁、跨项目 | 只有 ACTIVE 对象可回源，缺失/篡改可检测 |
| V5 迁移 | 数量、旧 ID、废弃保号、已知缺口、重复包 | 基线全对账，重复导入无新增正式事实 |
| 谱系与检索 | `结论 -> 审批 -> H -> U -> Q -> 原件`、权限和索引水位 | 已批准关键项 100% 可达，搜索先回权威库复核 |
| 身份与权限 | OIDC、MFA、CSRF、撤权、服务账号、Scope、直接 SQL | 无会话固定、跨租户、对象枚举或服务账号提权 |
| MCP/CLI/Skills | OAuth、stdio、工具表、钥匙串、Skill 静态规则 | 三入口状态版本一致，禁用工具不可调用，Skill 无事实/密钥 |
| Web/PWA | 响应式、键盘、WCAG、状态/错误、离线缓存与冲突 | 关键旅程桌面/手机可用；离线无正式写 |
| Dify | 重试、超时、取消、错误 Schema、外发、许可形态和降级 | 只生成 Proposal；停用后核心查询/审批完整 |
| Open Notebook/Nubase/FlowLong/Zvec | 隔离、权限、许可、重建、回调、退出 | 任一组件删除或禁用不破坏权威状态 |
| 安全 | 注入、伪 FINAL、路径穿越、压缩炸弹、SSRF、密钥泄漏 | 无权限提升、跨项目泄漏或正式状态污染 |
| E2E | 团队登录、会议、并发审批、撤权、证据、断网、恢复 | F4.8 与 F6.4 发布旅程全部通过 |
| 运维 | 连接池耗尽、磁盘满、进程/主机/外部组件故障、PITR | 核心按设计降级，告警及时，恢复满足目标 |

## 团队服务器试运行版验收

1. 新 AI 无聊天历史时，5 分钟内通过远程 MCP/API 说清项目阶段、已批准方向、开放项、当前任务、禁区和证据水位。
2. 任一已批准关键结论完成“正式状态 -> 审批事件 -> H -> U -> Q -> S3 原件版本”的回源，完整率 100%。
3. 开放讨论、模型候选、Dify 输出和外部工作流回调不会自动成为决定、约束、负责人、截止时间或提交版本。
4. 相同业务命令并发重试 100 次只产生一个事件；20 个并发审批只形成一个满足策略的最终状态。
5. 普通成员经 API、直接数据库尝试、搜索、对象 URL、导出和 Task Packet 均无法访问其他工作空间/项目。
6. 撤销成员后，Web、CLI、MCP 和自动化凭据在目标时间内失效；旧会话不能复活权限。
7. PWA 断网后只能看带时间戳快照和保存草稿，不能批准、删除原件或管理成员；联网冲突不会静默覆盖。
8. Zvec、Dify、Open Notebook、Nubase 或 FlowLong 任一停止时，权威状态、证据读取和核心人工审批仍可用。
9. 独立环境完成 PostgreSQL PITR、对象版本恢复、投影/索引重建、事件链与哈希校验，满足 `RPO <= 5 分钟`、`RTO <= 60 分钟`。
10. 团队成员一分钟内找到当前任务、会议、提交件和待确认项；单场会议默认不超过 8 张高价值审批卡，处理时间不超过 10 分钟。

## 服务目标与告警

| 指标 | 首个生产目标 | 告警/阻断 |
|:---|:---|:---|
| 核心 API 月可用性 | 首发 `>= 99.5%`；升级双节点和高可用数据库后 `>= 99.9%` | 错误预算快速消耗或连续 3 分钟核心不可用告警 |
| 读取 API P95 | `<= 500 ms`，不含 AI 生成 | 连续 10 分钟超标 |
| 非 AI 写入 P95 | `<= 1 s` | 连续 10 分钟超标 |
| Outbox/索引追平 P95 | `<= 5 s` | 最老消息超过 5 分钟告警 |
| 权威回源完整率 | `100%` | 任一正式结论无法回源立即阻断 |
| 未授权状态升级 | `0` | 任一事件立即进入安全事故流程 |
| PostgreSQL RPO | `<= 5 分钟` | WAL 归档中断超过目标窗口 |
| 核心服务 RTO | `<= 60 分钟` | 恢复演练超时阻断生产准入 |
| 初始容量 | 50 在线会话、10 并发写 | 压测未达标阻断发布或先降容量承诺 |

## 备份、恢复与发布门

- PostgreSQL 开启连续 WAL 归档和 PITR；每日快照与周期性逻辑导出位于不同故障域。
- 对象存储开启版本控制、加密和删除保护；关键原件可启用对象锁。
- 每月在隔离环境执行数据库、对象和配置联合恢复；每季度执行故障切换/主机丢失演练。
- 恢复顺序：冻结写入 -> 新实例 PITR -> 对象版本与哈希校验 -> 投影重建 -> 索引重建 -> 外部协调对账 -> 金标/权限烟测 -> 切换入口。
- 镜像固定版本和摘要，生产禁用 `latest`；数据库迁移显式执行并用 advisory lock 保证单实例。
- 发布采用 expand-migrate-contract；破坏性 contract 至少延迟一个发布周期；应用可快速回滚，数据库默认向前修复。
- 许可证、SBOM、依赖扫描、密钥扫描、金标、RLS、并发、备份恢复、安全扫描和组件降级必须全部通过。
