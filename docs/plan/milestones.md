# 里程碑

## 排期口径

- Phase 1-4 共 29 项，组成“团队服务器试运行版”。三条主 Lane 持续并行、具备独立基础设施和验收负责人时，预计 5-7 周。
- 单人顺序实施 Phase 1-4 预计 12-16 周；完整 42 项还需增加外部 POC、生产演练和 10-30 天自然试运行，应在 Phase 5 取得实测结果后重估。
- Phase 5-6 不与试运行版强绑。任何外部组件只有通过许可、隐私、真相、故障和退出门后才进入生产。
- 当前没有应用代码，所有日期应在 Phase 1 开工、人员与环境确认后排入迭代，不以本文档中的估算替代负责人承诺。

## M1：权威与安全契约冻结

**包含任务**：F1.1-F1.6。  
**目标**：在写代码前固定唯一权威、不可变原件、无正式离线写、租户角色、数据分级和跨入口契约。

### 完成标准

- 鸿日 V5 金标数量、固定查询、缺口和禁止状态升级反例可自动校验。
- ADR 接受 PostgreSQL 唯一正式写入、S3 不可变原件、SQLite 仅开发/测试/演示/只读快照，以及首发与高可用拓扑。
- 领域 Schema、稳定 ID、事件、状态机、端口、OpenAPI/MCP、幂等和 409 并发语义完成版本化。
- OIDC、MFA、RLS、租户上下文、角色、P0-P3 分级和威胁模型形成可测试契约。
- 工程、测试、密钥、日志、SBOM 和许可基线可运行；Dify 和 FlowLong 的附加许可门被明确登记。

### 阻断条件

- 仍允许本地 SQLite 与服务器同时写正式状态。
- AI、MCP、Skill 或外部工作流拥有批准能力。
- 生产 SLO、RPO/RTO、许可或数据外发责任没有明确所有者。

## M2：权威数据可恢复

**包含任务**：F2.1-F2.7。  
**目标**：形成 PostgreSQL 强一致状态、S3 原件谱系、可重建检索与经过演练的恢复链。

### 完成标准

- Alembic 显式迁移、租户复合外键和强制 RLS 通过跨租户测试，运行时角色无 `BYPASSRLS`。
- 业务写入、事件、最小投影、审计关联和 Outbox 同事务提交；幂等重试只产生一个事件，版本冲突返回 409。
- 原件经过隔离、哈希/MIME/安全校验后按 SHA-256 寻址；只有 ACTIVE 对象可成为证据。
- V5 全量数量、旧 ID、废弃保号和已知缺口对账，重复导入结果不变。
- PostgreSQL FTS/`pg_trgm` 基线支持权限过滤、索引水位和稳定 ID 回源；Outbox Worker 可重复投递、死信和世代重建。
- 独立故障域备份和隔离环境联合恢复满足 `RPO <= 5 分钟`、`RTO <= 60 分钟`，并通过事件链、对象哈希、权限和金标校验。

### 阻断条件

- 备份未在独立环境恢复，或对象版本窗口短于 PostgreSQL PITR 窗口。
- 搜索、Worker 或对象存储故障可产生“业务成功但无事件/无可回源原件”。
- 应用启动自动修改生产 Schema。

## M3：团队与 AI 统一受控访问

**包含任务**：F3.1-F3.8。  
**目标**：让具名成员和受限 AI 通过统一服务工作，确保跨入口权限、状态版本和证据一致。

### 完成标准

- OIDC、MFA、邀请、会话轮换、撤权、服务账号和 RLS 事务上下文通过安全测试。
- 组织/项目/对象/动作授权和保密策略同时在应用层与数据库层生效；连接池复用不串租户。
- Proposal、批准、修改、驳回、替代、重开和并发审批形成完整事件与审计链；AI 不能批准。
- Task Packet 只含任务所需状态、证据、开放项、禁区、版本和事件水位，关键结论可回源。
- HTTP/OpenAPI、远程 Streamable HTTP MCP、本地 stdio 代理、CLI 和 Skills 调用同一应用用例，响应状态版本一致。
- MCP 工具允许列表不包含批准、任意 SQL、成员管理或硬删除；CLI 凭据进入系统钥匙串；Skill 不保存业务事实或密钥。

### 阻断条件

- 撤权后任一 Web 会话、CLI Token、MCP OAuth 或服务凭据仍可在目标窗口外使用。
- 客户端或第三方组件直接持有生产数据库凭据。
- 同一项目经 Web、MCP 和 CLI 返回不同正式状态版本且无陈旧提示。

## M4：团队服务器试运行版

**包含任务**：F4.1-F4.8。  
**目标**：交付可远程协作、可监控、可恢复的响应式 Web/PWA 以及会议到人工批准的日常闭环。

### 完成标准

- 会议原件先准入 S3，直接 Worker 生成带原文、上下文、来源和运行版本的候选，不自动批准。
- Web/PWA 提供今日、工作、知识、待确认、AI 工作和管理入口；关键桌面/手机路径满足 WCAG 2.2 AA。
- 上传、检索、证据回源、审批差异、409 冲突、团队角色、审计、系统健康和 AI 连接可完成真实操作。
- 离线仅允许带过期提示的只读快照和草稿；P2/P3 默认不持久缓存；离线不能批准或修改成员。
- 公网仅开放 HTTPS 443；`livez/readyz`、监控、告警、迁移锁、固定镜像摘要和回滚演练通过。
- 跨租户泄漏和未授权升级为 0；20 个并发审批只产生一个合法结果；关键结论 100% 回源；撤权、断网、PITR、对象恢复、V5 金标和新 AI 5 分钟冷启动旅程全部通过。

### 阻断条件

- 单场会议默认高价值审批卡超过 8 张或处理时间超过 10 分钟，且没有先优化聚类/风险分级。
- 派生组件故障使权威状态、证据读取或人工审批整体不可用。
- 恢复、租户隔离、撤权或并发验收任一失败。

## M5：五组件准入结论

**包含任务**：F5.1-F5.7。  
**目标**：用同一契约、金标和故障框架评估五个外部组件，输出明确采用、推迟或拒绝结论。

### 完成标准

- 统一评分覆盖回源、中文、权限、隐私、许可、质量、性能、可维护性、恢复和退出，不以星数作为采用依据。
- Zvec 只由单活动 Worker 写入，检索命中回 PostgreSQL 复核；可删除重建并与 PostgreSQL 基线 A/B。
- Open Notebook/`content-core` 只单向处理原件副本，研究输出只生成 Proposal，关闭后主系统完整。
- Nubase 仅按 Auth/Storage/Gateway/Memory 单项评估，不替代标准 PostgreSQL PITR/HA，也不保存正式状态。
- FlowLong 只有书面许可和复杂人工流程需求同时成立才运行真实 POC；AI 审批与超时自动通过强制关闭。
- Dify 限单内部 Workspace、后端 API 和最小 Task Packet；输出经 Schema 校验只生成 Proposal；源码多租户、白标或分发必须先获书面授权。
- 五份结论合并为可审计 ADR，逐项包含版本、许可、数据流、真相边界、SLO、故障行为和退出方案。

### 阻断条件

- 任一 POC 修改领域 Schema、获得正式写权限、共享数据库管理员或对象存储管理员凭据。
- 未通过许可/隐私门就使用真实数据部署 Dify 或 FlowLong。
- 组件无法禁用、导出或从权威数据重建，却被建议采用。

## M6：生产准入与第二项目证明

**包含任务**：F6.1-F6.6。  
**目标**：将通过 ADR 的组件纳入可降级架构，并以真实负载、恢复、安全和长期团队使用证明生产可行性。

### 完成标准

- 所有选中组件具备 NoOp、功能开关、固定版本、基线实现和退出测试；任何单组件停止不影响权威读取与人工审批。
- ModelGateway/AIWorkflow 记录 Task Packet、模型/工作流版本、数据级别、授权、成本和最终 Proposal；P2/P3 默认不外发。
- BrandBench 含 12-20 个固定任务及提示注入、伪 FINAL、无证据、跨租户和误升级样本；硬失败自动淘汰。
- 50 个在线会话、10 个并发写请求、派生组件中断、连接池/磁盘/主机故障、PITR、对象恢复、密钥轮换和安全发布演练达标。
- 鸿日完成 5-10 个真实任务和 10-30 天团队试运行，权威回源 100%、未授权升级 0，审批与重复解释指标有可复核变化。
- 第二项目完成独立冷启动、RLS、对象、索引、Task Packet、外部流程、备份和容量验证，不复制鸿日业务事实。

### 阻断条件

- 负载、恢复、安全发布或密钥轮换任一无法达到既定 SLO。
- 真实试运行无法维持 100% 回源或出现未经授权状态升级。
- 第二项目隔离失败，或容量/成本没有测量就宣称可生产扩展。
