## M2：服务器权威基础

**状态**：已完成。

### 完成条件

- PostgreSQL 是团队唯一可写正式状态源；
- S3 兼容对象存储按 SHA-256、版本和 ACTIVE 状态管理原件；
- 通用 OIDC、项目 RBAC/RLS、幂等、乐观锁、审计和 Outbox 通过；
- 版本化 HTTP API、OpenAPI、健康、指标、告警和共享限流通过；
- 空库恢复、事件重建投影和 S3 VersionId 对账通过。

### 保留限制

- F2.4 只证明通用 OIDC 协议与会话实现，不代表 Den 已经接入；Den 发行者、第一方受众和撤权联动由 F3.5 完成。
- F2.10 逻辑备份不是生产 PITR；批准的 99.5%、PostgreSQL RPO<=5 分钟、核心 RTO<=60 分钟仍需 M4 实测。

## M3：Den 与联网产品

**状态**：进行中，F3.1/F3.2 已完成，当前 F3.3。

### 已完成

- SQLite 到 PostgreSQL/S3 一次性迁移和写入冻结通过；
- Den API/Web 生产构建、MySQL bootstrap、单组织注册、桌面交接通过；
- 普通员工可通过 Den 使用组织共享 Skill、共享模型和公司 MCP；
- 模型和 MCP 撤权后立即不可见、不可调用；
- 许可证结论固定为 `FSL-1.1-MIT` 内部自托管，不再称为开源 Den。

### 完成条件

1. Den Web/API/MySQL 和远程 Worker 有可重复的公司部署、备份恢复、受控内网 HTTP/HTTPS 入口、监控和升级回滚流程。
2. 普通员工在 FoxWork 自助注册并登录 Den 后进入唯一公司组织，不能创建第二组织，不再看到第二套账号或旧团队连接页。
3. FoxWork、Den 员工页面和 Den 后台管理员网页全部为自然简体中文，没有语言切换、英文回退或内部键名。
4. Brand OS 使用 Den OAuth/OIDC 第一方短期令牌，严格校验受众和 `(issuer, subject)`；可信 Den 成员首次登录可建立内部身份映射，但不自动获得项目权限，成员撤权在业务 API、MCP、模型和 Skills 上收敛。
5. Den 组织/团队/远程工作区与 Brand OS 项目通过显式映射连接，一次登录只显示获授权运行空间和项目。
6. 员工可上传图片、视频、录音、PPT、Office 和 PDF；可查看准入、解析、转写和分析进度。
7. 每个解析结果可回到原件版本、页码/幻灯片/时间码；模型结果只能形成 Artifact 或 Proposal。
8. 当前状态、证据、变化、Proposal 差异和并发冲突可在 FoxWork 中完成。
9. Brand OS MCP 登记到 Den，员工通过 Agent MCP 搜索和执行；无人工批准工具、任意 SQL 或任意文件工具。
10. 公司 Skills、共享模型和桌面策略按成员/团队下发，撤权和回滚通过。
11. Dify 通过 `AIWorkflowPort` 接入；Zvec、Open Notebook、Nubase、FlowLong 各自形成采用、隔离试用或拒绝结论。
12. F3.19 在测试环境通过 G-01-G-24、多客户端、远程 Worker、长任务、离线/恢复、安装烟测、安全、品牌、凭据和中文产物扫描；通过后统一同步 Wiki。

### 阻断条件

- 把 Den MySQL、Worker 文件系统或 OpenWork Session 当正式状态源；
- 复用 Den Session Token 访问 Brand OS，或让共享服务令牌冒充员工；
- 注册后可创建第二组织、需要第二套密码、撤权后仍可使用能力；
- 视频/音频/PPT 等结果无法回到来源定位；
- FoxWork、Den 员工页或后台管理员网页出现英文回退、上游品牌、未解析键名或开发占位文案；
- Den 远程工作区/Worker 未在测试环境真实运行，或其运行数据被当作业务真相；
- 真实资料在安全、构建和产物门通过前接入。

## M4：团队生产准入

**状态**：待开始。

### 完成条件

- 试点成员、Den 角色、项目权限和资料范围获 Fox 确认；
- 自助注册、第二组织拒绝、登录、重验、离职撤权、账号恢复和角色变化全链通过；
- 两个以上客户端并发写入无丢失、无双主、无静默覆盖；
- 至少两轮真实会议和一轮图片/视频/录音/PPT 工作形成可回源结果；
- BrandBench、多模型、外发、安全、供应链和许可证检查通过；
- Den MySQL/远程 Worker、Brand OS PostgreSQL/S3、API、资料 Worker 和队列完成隔离恢复演练；
- 实测已批准的 99.5% 月可用性、PostgreSQL RPO<=5 分钟、核心 RTO<=60 分钟；Den MySQL/Worker 的 RPO/RTO 或运行目标由 Fox 另行批准并记录；
- FoxWork 完成签名、公证、内部更新、灰度和坏版本回滚；
- Fox 明确给出 Go、延长或 No-Go；通过后统一同步 Phase 4 Wiki。

### 阻断条件

- 账号或项目越权、撤权延迟超过批准窗口、敏感令牌进入日志；
- 任一数据库恢复后与审计、事件、投影或原件版本不一致；
- 解析服务故障导致正式状态损坏或原件丢失；
- 未签名包、上游更新源或非中文构建向员工分发；
- 用供应商宣传或一次本机演练代替真实 SLO 证据。
