# 任务分解

> 追踪模式：`LOCAL_ONLY`  
> 总计：5 个阶段，56 项；已完成 29 项，当前 29/56。

## 任务定义

交付 FoxWork：公司定制版 OpenWork 是员工唯一客户端，普通员工在公司 Den 入口自助注册并登录一套账号后，直接进入唯一公司组织和获授权的远程工作区，获得按成员或团队下发的模型、MCP 和 Skills，并访问获授权的品牌项目工作区。Brand Project OS Service 负责项目资料、正式状态、证据、Proposal、人工确认和视频/录音/图片/PPT/Office/PDF 分析，不形成第二个客户端或第二套账号。

## 全局架构约束

- **S**：Den 只做账号与 AI 控制面；Brand OS 只做品牌项目业务；FoxWork 只做员工交互和本机桥接。
- **U**：FoxWork/Agent -> 版本化 API 或受控 MCP -> 应用用例 -> 权威端口；外部组件不能反向修改正式表。
- **P**：身份、项目、上传、处理任务、Artifact、MCP、Skill、模型和工作流都先冻结版本化 Schema。
- **E**：本机、测试和生产只通过配置变化；FoxWork 只配置 Den Web 公共地址，不内置 API、数据库或对象存储地址。
- **R**：Den、模型、解析、检索、记忆、工作流和 Worker 都通过端口隔离；Den 远程 Worker 必须完成运行验收，停用其他可选适配器后，正式状态和原件仍完整。

每项实现必须增加与风险相称的自动化测试。纯文档任务需运行链接、术语和旧约束扫描。稳定规则写入 `AGENTS.md` 和 ADR；当前没有获准的仓库内 Memory 文件。
