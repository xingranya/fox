# Phase 2：权威数据与可靠性脊柱

**目标**：建立 PostgreSQL 权威事务、对象证据、可靠异步派生、可回源检索和可演练恢复能力。  
**状态**：未开始  
**计划窗口**：Phase 1 契约稳定后约 2 周，可与 Phase 3 的非依赖工作交叠

## 任务

- [ ] **F2.1：实现 PostgreSQL Schema、Alembic、租户字段与强制 RLS**
  - 优先级：P0
  - 工作量：L
  - 依赖：F1.3、F1.4
  - Lane：A
  - 测试期望：增加空库升级、历史版本升级、回滚边界、租户必填、RLS 跨租户拒绝和连接池上下文清理测试。
  - 治理影响：固定迁移编号所有权、生产迁移必须显式执行、所有权威表必须带租户边界的规则。
  - 验收：同一迁移可重复部署到干净环境；应用启动不自动修改生产 Schema；无租户上下文或跨租户查询默认拒绝。
  - 备注：无。

- [ ] **F2.2：实现命令事务、事件、同步最小投影、幂等、乐观锁与 Outbox**
  - 优先级：P0
  - 工作量：L
  - 依赖：F2.1、F1.5
  - Lane：A
  - 测试期望：增加事务回滚、重复幂等键、版本冲突、并发审批、Outbox 同事务落盘、崩溃重启和投影回放测试。
  - 治理影响：固定“命令结果、事件、同步投影、Outbox 同事务提交”和事件只追加规则；冲突不得最后写入覆盖。
  - 验收：任一写入失败不会留下部分正式状态；重复请求返回同一语义结果；旧版本写入返回 409；投影可从权威事件重建。
  - 备注：无。

- [ ] **F2.3：实现 S3/MinIO 隔离区、校验、内容寻址与版本谱系**
  - 优先级：P0
  - 工作量：L
  - 依赖：F1.3、F1.4
  - Lane：B
  - 测试期望：增加重复内容、改名、伪造 MIME、超限文件、校验失败、隔离区转正、缺失对象和跨租户对象访问测试。
  - 治理影响：固定原件、提交件、母稿、基线和派生稿的权威等级；对象键不得直接暴露业务机密或成为独立真相源。
  - 验收：文件先进入隔离区并完成哈希与格式校验；SHA-256 相同内容不重复存储；版本谱系可从业务记录回到原始对象且不静默覆盖。
  - 备注：无。

- [ ] **F2.4：实现 V5 幂等导入、旧 ID 保真与数量对账**
  - 优先级：P0
  - 工作量：M
  - 依赖：F2.2、F2.3
  - Lane：B
  - 测试期望：使用 F1.1 金标增加数量对账、重复导入、坏行、废弃保号、缺失源、断点续传和部分失败回滚测试。
  - 治理影响：记录 V5 字段映射、不可修复缺口、导入批次和操作者；禁止根据缺失字段自动补造事实。
  - 验收：重复导入不产生副本；旧 ID、废弃记录和替代关系保留；每批导入有数量、哈希、错误清单和可重放审计记录。
  - 备注：受 F1.1 的 V5 数据副本阻塞影响。

- [ ] **F2.5：实现 PostgreSQL FTS、`pg_trgm`、可选 pgvector 与稳定 ID 回源**
  - 优先级：P0
  - 工作量：M
  - 依赖：F2.4
  - Lane：C
  - 测试期望：增加中文关键词、模糊匹配、过滤、排序、权限裁剪、向量关闭降级和从命中项到原件的全链回源测试。
  - 治理影响：固化“检索结果只是候选，权威结论必须回到事件与原件”；pgvector 仅在基准收益成立后启用。
  - 验收：结果包含稳定 ID、原话、上下文、来源和权限信息；关闭向量能力仍能检索；常用查询达到约定秒级目标且无跨租户命中。
  - 备注：无。

- [ ] **F2.6：实现 Outbox Worker、Inbox 去重、有限重试、死信与索引世代**
  - 优先级：P0
  - 工作量：L
  - 依赖：F2.2、F2.5
  - Lane：C
  - 测试期望：增加重复消息、乱序、超时、进程崩溃、毒消息、退避重试、死信处理、全量重建和世代切换测试。
  - 治理影响：固定至少一次投递与消费者幂等规则；所有检索索引和外部工作流只消费派生事件，不得反向覆盖正式状态。
  - 验收：重复或乱序消息不破坏结果；失败可观测、可重试、可隔离；新索引世代完整后原子切换，并显示最新处理位点或延迟。
  - 备注：无。

- [ ] **F2.7：建立 PITR、对象版本、跨故障域备份、联合恢复、重建与诊断**
  - 优先级：P0
  - 工作量：L
  - 依赖：F2.3、F2.6
  - Lane：D
  - 测试期望：执行时间点恢复、误删对象恢复、空环境恢复、损坏投影重建、索引重建、缺失备份和校验失败演练。
  - 治理影响：固定备份保留、加密、访问责任、跨故障域和定期恢复演练规则；备份成功不等于恢复可用。
  - 验收：在隔离环境完成数据库与对象联合恢复；金标查询与关键哈希一致；演练达到 RPO 不超过 5 分钟、RTO 不超过 1 小时的初始目标，差异不会被静默修复。
  - 备注：无。

## 阶段备注

- 数据库迁移与权威事务只由 Lane A 编号和合并。
- F2.1 与 F2.3 可并行；F2.5 完成后，F2.6 与 F2.7 形成可靠派生和恢复链。
- PostgreSQL 是正式状态唯一权威库；S3/MinIO 保存原件，二者通过稳定 ID、哈希和事务记录关联。
