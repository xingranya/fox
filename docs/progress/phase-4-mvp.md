# Phase 4：Web/PWA 与团队服务器试运行版

**目标**：交付团队可共同使用的工作型前端、会议增量闭环、服务器部署运维和金标端到端旅程。  
**状态**：未开始  
**计划窗口**：Phase 1-4 采用三条执行 Lane 总计 5-7 周；本阶段约占最后 2-3 周

## 任务

- [ ] **F4.1：实现会议准入、分段、信息候选与直接 Worker**
  - 优先级：P0
  - 工作量：M
  - 依赖：F2.3、F2.6、F3.3
  - Lane：A
  - 测试期望：增加缺录音、说话人、时间戳、重复上传、恶意文件、分段边界、分类反例、Worker 重试和候选幂等测试。
  - 治理影响：固定会议材料完整性、置信度和候选类型规则；直接 Worker 是可控基线，任何 AI 输出都只进入 Proposal。
  - 验收：会议候选保留原文、上下文、说话人、时间、置信度和来源；重复处理不产生副本；Worker 失败可重试且不会自动批准。
  - 备注：Dify 在 F5.6 才与本直接 Worker 做 A/B，不阻塞试运行版。

- [ ] **F4.2：实现响应式 Web/PWA 壳、导航、身份状态与无障碍基线**
  - 优先级：P0
  - 工作量：L
  - 依赖：F3.6
  - Lane：B
  - 测试期望：增加路由、登录/登出、会话过期、权限导航、键盘操作、焦点管理、响应式布局和 WCAG 2.2 AA 自动检查。
  - 治理影响：前端所有文案均面向真实用户；状态不能只依赖颜色；权限控制以服务端为准，前端只做可用性约束。
  - 验收：应用直接进入工作界面；导航包含今日、当前状态、待我确认、会议与变化、证据检索、任务交付、研究工作台和系统健康；桌面与移动视口无重叠。
  - 备注：无。

- [ ] **F4.3：实现今日、工作、知识、检索、上传与证据详情**
  - 优先级：P0
  - 工作量：L
  - 依赖：F4.2、F3.5、F2.5
  - Lane：B
  - 测试期望：增加组件、API 集成、空/错/加载状态、权限裁剪、检索回源、上传进度、重复文件和响应式 E2E 测试。
  - 治理影响：界面区分正式状态、候选和派生摘要；上传先进入隔离区；任何摘要都提供回源入口。
  - 验收：成员一分钟内看到阶段、本人任务、开放项和变化；搜索结果可打开证据原件与上下文；上传失败可恢复且不会重复创建正式记录。
  - 备注：无。

- [ ] **F4.4：实现待确认、会议变化、审批差异与并发冲突处理**
  - 优先级：P0
  - 工作量：L
  - 依赖：F4.1、F4.3、F3.4
  - Lane：B
  - 测试期望：增加批准、修改、驳回、替代、重开、批量边界、双人并发、409 冲突刷新、撤权和完整审批 E2E 测试。
  - 治理影响：批准操作必须展示证据、影响范围和当前版本；冲突不能静默覆盖；高风险批量批准默认禁止。
  - 验收：批准前正式状态不变；冲突时保留用户输入并要求基于最新版本重新确认；每次操作可追溯到人、理由、证据和版本。
  - 备注：无。

- [ ] **F4.5：实现团队角色、审计、系统健康与 AI 连接管理**
  - 优先级：P0
  - 工作量：M
  - 依赖：F3.2、F4.2
  - Lane：C
  - 测试期望：增加成员邀请/撤权、角色变更、审计过滤、健康状态、MCP/模型连接配置、密钥遮罩和无权限访问测试。
  - 治理影响：管理面与业务审批面分离；密钥只由服务端安全保存；健康页不得泄漏内部路径、令牌或敏感正文。
  - 验收：Owner 可管理成员与连接，Approver 不能越权管理基础设施；审计可按人、项目、动作和关联 ID 查询；关键依赖状态和最近成功时间可见。
  - 备注：无。

- [ ] **F4.6：实现 PWA 离线只读、草稿、过期提示与联网冲突处理**
  - 优先级：P0
  - 工作量：M
  - 依赖：F4.3、F4.4
  - Lane：C
  - 测试期望：增加断网读取、敏感缓存限制、草稿本地保存、缓存过期、恢复上线、令牌撤销、版本冲突和设备清理测试。
  - 治理影响：离线数据不是正式状态；禁止离线批准或静默队列化正式写入；缓存范围由保密级别与设备策略控制。
  - 验收：断网时明确标识数据时间和只读状态；用户可保存未提交草稿；联网后必须重新鉴权、获取最新版本并处理冲突后才能提交。
  - 备注：无。

- [ ] **F4.7：建立服务器部署、TLS、健康检查、监控、告警与回滚**
  - 优先级：P0
  - 工作量：L
  - 依赖：F2.7、F3.6
  - Lane：D
  - 测试期望：增加部署配置校验、TLS、启动探针、就绪探针、依赖故障、日志指标、告警路由、版本升级和回滚烟测。
  - 治理影响：固定生产变更审批、迁移先后顺序、告警责任和回滚手册；首发可用 Docker Compose，不把 Kubernetes 设为前置条件。
  - 验收：全新服务器可按手册部署；HTTPS 强制启用；API 与 Worker 独立健康可见；首发月可用性目标为 99.5%，达到升级门后双节点目标为 99.9%；失败发布可回滚到上一版本且不逆向破坏 Schema 或权威数据。
  - 备注：单服务器仍是已接受的首发单点，后续按 SLO 决定托管数据库或多节点升级。

- [ ] **F4.8：执行租户、并发、撤权、断网、恢复、金标与用户旅程验收**
  - 优先级：P0
  - 工作量：XL
  - 依赖：F4.4、F4.5、F4.6、F4.7
  - Lane：D
  - 测试期望：运行跨租户攻击、双人并发审批、即时撤权、Worker 重放、断网恢复、PITR 恢复和鸿日 V5 金标全链 E2E。
  - 治理影响：把偏差、SLO 实测、人工审批负载和残余风险写入进度与 ADR；未通过硬门不得进入外部组件 POC。
  - 验收：跨租户泄漏和未经批准升级均为 0；20 个并发审批只落一个合法结果；撤权覆盖 Web、CLI、MCP；断网不产生正式写；恢复后金标一致且满足 RPO/RTO；关键结论 100% 回源；新 AI 5 分钟内取得正确 Task Packet。
  - 备注：F1.1 数据副本未提供时，鸿日真实金标部分无法完成。

## 阶段备注

- Lane A 建立会议直接 Worker；Lane B 负责主工作流页面；Lane C 负责管理面与离线边界；Lane D 负责部署和最终验收。
- Web/PWA 是首发主客户端；桌面壳后置，不作为试运行版前置条件。
- Phase 4 通过即形成 29 项团队服务器试运行版，之后才允许启动隔离 POC。
