# Phase 1：契约、身份与安全基线

**目标**：在实现任何正式写入前，冻结服务器权威、领域语义、租户边界、身份安全和版本化接口。  
**状态**：未开始，等待用户确认执行  
**计划窗口**：三条执行 Lane 并行时约 1 周

- [ ] **F1.1：固定鸿日 V5 黄金样本、验收查询和禁止升级反例**
  - 优先级：P0
  - 工作量：S
  - 依赖：无
  - Lane：A
  - 测试期望：增加 Fixture 完整性、固定查询快照、数量基线、缺证据和越权状态升级反例测试。
  - 治理影响：固定样本保密级别、脱敏边界和“缺证据不硬猜”规则；稳定规则写入 ADR 或共享指令面。
  - 验收：固定 19 源、4903 原始段、4180 业务段、762 证据卡、690 有效、72 废弃保号、18 档案和 565 议题簇；缺源标记 `KNOWN_SOURCE_GAP`；覆盖冷启动、版本、回源、越权升级和增量会议。
  - 备注：当前工作区缺少鸿日 V5 数据副本，本任务被阻塞。

- [ ] **F1.2：接受服务器权威、无正式离线写、部署拓扑与 SLO ADR**
  - 优先级：P0
  - 工作量：M
  - 依赖：无
  - Lane：B
  - 测试期望：文档与配置静态校验；用故障场景表验证单服务器、数据库不可用、网络中断和恢复边界。
  - 治理影响：明确服务器是唯一正式写入入口，`LOCAL_ONLY` 仅为任务追踪；记录 RPO、RTO、责任人和生产迁移规则。
  - 验收：ADR 明确 PostgreSQL、S3/MinIO、API、Worker、Web/PWA 的职责；离线客户端只能读取缓存或保存未提交草稿；首发月可用性目标为 99.5%，升级双节点后为 99.9%，RPO 不超过 5 分钟、RTO 不超过 1 小时。
  - 备注：无。

- [ ] **F1.3：定义领域 Schema、稳定 ID、事件、状态机和版本化端口**
  - 优先级：P0
  - 工作量：L
  - 依赖：F1.1、F1.2
  - Lane：A
  - 测试期望：增加 JSON Schema 正反例、旧版本兼容、非法状态迁移、事件回放和端口共享契约测试。
  - 治理影响：把正式状态不变量、端口版本策略、弃用窗口和 `supersedes` 规则写入 ADR 与共享指令面。
  - 验收：`VIEW`、`HYPOTHESIS`、`OPTION`、`OPEN` 不能直接升级为正式决定或约束；稳定 ID、废弃保号、替代关系和跨层序列化契约明确；领域核心不依赖具体外部组件。
  - 备注：无。

  - 治理影响：确定 Owner、Approver、Contributor、Viewer 的责任边界，以及组织、工作区、项目三级数据所有权和外发规则。
