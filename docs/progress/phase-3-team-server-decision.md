**目标**：在本地价值被证明后，依据真实团队需求决定保持本地、局部共享或启动独立团队服务器实施 SPEC。
**状态**：待开始
**任务数**：7
**自适应阈值**：标注 2 / 重计划 3 / 重定范围 5

## 任务

- [ ] **F3.1：基于 Phase 2 证据确认团队用户、协作场景及 FoxWork 是否合线**
  - 优先级 / 工作量 / Lane：P0 / M / Gate
  - 依赖：F2.8=Go
  - S.U.P.E.R：S、U、P
  - 测试期望：访谈记录、场景覆盖、指标链接和决策表静态校验；不新增运行时行为。
  - 治理影响：明确需求所有者、用户、共享/私有边界与不合线项。
  - 验收：回答谁需要同时使用、哪些动作协作、哪些数据共享、FoxWork 是否合线；输出继续本地/局部共享/团队服务器三选一门，不因“未来会用”自动通过。
  - 备注：本任务是 Phase 3 总开关。

- [ ] **F3.2：评估本地/服务器数据边界及 PostgreSQL、S3、同步迁移候选**
  - 优先级 / 工作量 / Lane：P1 / L / A
  - 依赖：F3.1 允许评估
  - S.U.P.E.R：U、P、E、R
  - 测试期望：数据迁移/回退演算、SQLite 导出、冲突模型、原件哈希和恢复 POC 计划校验。
  - 治理影响：候选不是已批准架构；记录数据真相、迁移和退出条件。
  - 验收：比较继续 SQLite、本地+只读同步、PostgreSQL+S3；列出数据量、并发、离线、迁移、成本、RPO/RTO 和回退证据；未证明需要则保持本地。
  - 备注：不直接部署生产数据库或对象存储。

- [ ] **F3.3：评估 OIDC、RBAC、RLS、审计、并发一致性与 HA/灾备需求**

  - 测试期望：角色/动作矩阵、威胁模型、并发场景、撤权和恢复演练计划审查。
  - 治理影响：形成身份/权限/运维候选与明确非目标。
  - 验收：只为已确认多人动作设计身份/权限/审计；比较简单账号/OIDC、应用授权/RLS、单节点/HA；定义幂等、乐观锁、Outbox 和 PITR 的触发条件。
  - 备注：无触发证据时不实施企业体系。
