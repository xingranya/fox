# Phase 4：团队试点与生产准入

**目标**：用真实团队工作验证统一账号、资料分析、业务一致性、稳定性、安全和实际提效，再决定生产范围。  
**状态**：待开始，0/10。  
**前置**：F3.19 联网产品门通过。

## 任务

- [ ] **F4.1：确认试点成员、Den 角色、项目权限和资料范围**
  - 验收：Fox 批准成员、角色、项目和文件清单；不自动接管 NAS、个人工作区或团队全部文件。

- [ ] **F4.2：验证账号完整生命周期**
  - 测试：自助注册、第二组织拒绝、登录、重验、角色变化、离职撤权、旧 Session/MCP Token、远程工作区、模型/Skill 授权和账号恢复。
  - 验收：一处撤权在所有入口生效，不留下孤立 Brand OS 账号。

- [ ] **F4.3：验证多客户端并发、幂等和正式状态一致性**
  - 测试：双写竞争、重试、陈旧缓存、断线重连和恢复。
  - 验收：无丢失、无静默覆盖、无双主。

- [ ] **F4.4：连续接入真实会议和多媒体资料并完成实际工作**
  - 测试：至少两轮真实增量和一轮图片/视频/录音/PPT；记录证据回源、错误和人工修正。
  - 验收：完成一次探索到执行闭环，处理结果被真实工作采用或有明确拒绝原因。

- [ ] **F4.5：复测多模型一致性和 BrandBench 人工质量**
  - 测试：同一 Packet、多模型盲评、版本、成本和一票否决。
  - 验收：模型切换不要求重讲背景，一票否决不能被平均分抵消。

- [ ] **F4.6：演练 Den、Brand OS、数据库、对象存储和队列中断与恢复**
  - 测试：Den API/Web/MySQL/远程 Worker、Brand OS API/PostgreSQL/S3、资料 Worker/队列、只读降级和全量对账。
  - 验收：账号控制面和业务权威分别恢复，缓存不冒充最新状态，原件和事件不丢失。

- [ ] **F4.7：身份、权限、审计、外发和供应链检查**
  - 测试：跨项目、撤权、令牌泄露、服务账号批准、日志脱敏、SBOM、漏洞和 FSL/MIT 归属。
  - 验收：高风险项关闭，服务账号、MCP、Skill、工作流无人工批准权。

- [ ] **F4.8：容量、性能、成本和 SLO 验证**
  - 测试：并发、远程 Worker、长媒体任务、限流、成本、告警和恢复时间。
  - 验收：实测已批准的 99.5% 月可用性、PostgreSQL RPO<=5 分钟、核心 RTO<=60 分钟；Den MySQL 与远程 Worker 指标单独获 Fox 批准。

- [ ] **F4.9：FoxWork 签名、公证、内部更新、灰度和回滚**
  - 验收：员工只维护一个 FoxWork；更新失败或坏版本能恢复上一内部版本。

- [ ] **F4.10：团队生产 Go、延长或 No-Go 与归档**
  - 验收：Fox 明确批准范围、SLO、责任人和已知限制；未通过则延长或停止，不虚报上线；通过后统一同步 Phase 4 Wiki。
